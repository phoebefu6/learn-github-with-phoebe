# Presenter notes - Session 1 · Meet GitHub

## Preflight (5 min before)
- Open courses/01-meet-github.html, Projector zoom 125%.
- Test the sorter widget once (one right, one wrong click).
- Be signed OUT of GitHub in one browser profile - you will demo the signup flow live.
- Have github.com/github/docs open in a tab (the tourist stop).
- Know your own username-repo trick result to show (your profile README).

## Run of show
- 0-3 · Welcome. Ask: "who has been sent a GitHub link and quietly panicked?" - hands go up, promise: 45 minutes from now that link is legible. State the no-code guarantee out loud.
- 3-8 · Part 1 what GitHub is. Lead with the Germany's-laws story - it lands harder with this audience than any software example. Then the three superpowers in office terms.
- 8-13 · Part 2 mental map. Read a URL aloud character by character: github.com / who / what. Do it twice with different repos. This is the highest-value 90 seconds of the session.
- 13-18 · Part 3 social layer. Show a good profile and an empty one side by side. Tease the username-repo easter egg but do not reveal the mechanics - Apply-along 3 is the payoff.
- 18-26 · Apply-along 1 account setup. The slowest step is always username choice - give the advice early (close to professional name) and set a 3-minute timer.
- 26-33 · Apply-along 2 sorter. Solo on devices. Item 5 (line comment) splits the room between Issue and PR - let the debate run one minute, the distinction IS the Session 4 preview.
- 33-40 · Apply-along 3 profile README. The "You found a secret!" banner gets an audible reaction - pause for it. Everyone commits two lines minimum; polish is Session 2's job.
- 40-45 · Q&A + quiz, show of hands per question.

## Never cut
- Reading URLs aloud (Part 2) - the orientation habit everything else builds on.
- The "You found a secret!" moment in Apply-along 3 - first commit, first repo, visible payoff.
- The commit-is-a-save-with-a-note framing - it carries Sessions 2 and 4.

## Cut if long
- Part 3 explore/trending goes to a self-study pointer.
- Apply-along 1 step 3 (github/docs tour) - compress to "star it from my screen, tour it at home".
- git vs GitHub card - point at the page.

## Expected questions
- "Is GitHub free?" - Yes for everything in this course. Paid tiers are for private-repo teams and enterprise controls.
- "Will I break something?" - You cannot break other people's repos; you can only propose. Your own repos keep full history - nothing is ever lost.
- "Why not Google Docs?" - Docs is great for co-writing one file. GitHub wins when work has structure: many pages, tasks, reviews, versions, publishing. Session 4 makes this visceral.
- "Do I need to learn the command line later?" - Only if you want the tech track. This track never touches it.
- "Is my company's code on here public?" - No - private repos and org permissions are the workplace default. Session 6 covers the safety basics.
